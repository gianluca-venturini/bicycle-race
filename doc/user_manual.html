User Manual

How to use this powerful tool

Map centered application

The aim of the project is displaying geographical data regarging Divvy bikes in the time period that goes from July to December 2013. For that reason evey data is geolocalized on the map that cover the entire surface of the application. An overlay layer of semi-transparent controls is present to let the user iteract with the application. Every control is present only when the user need it. In every moment the user can move the map, zoom in for a more detailed view or zoom out for see more general patterns.


Controls

Controls are the graphical elements that let the user interact. They are designed in a minimalistic style in order to provide only the useful functions in a intuitive and straightforward way. There are two different type of controllers:

  • Active controllers: that enable the user to request information (that the application will display in different way). They can be hidden when the user don't need them.
  • Enable controllers: this kind of controllers enable/disable the visualization of other controllers and for that reason are always visible.

Map layer control

This control enable the user to choose a type of map that he/she want to visualize. There are two buttons:

  • Satellitar map: visualize real photo taken from satellite.
  • Street map: visualize a cartographic reconstruction on Chicago.

    • Selection control

    This controller contains the buttons that enable the selection and deselection of all the stations in Chicago. There is also a button that enable/disable the visualization of the community areas present in city.

    Calendar control

    This control displays a calendar, the user can select a month from it clicking on the number of the mount (on the left of the controller). After selecting the month in the right part the controller enable the user to choose the day of the month. Once the user select both mount and day a new controller will appear on the screen: day controller

    In order to display and hide the calendar the user can press on the enable calendar control

    Day control

    This controller displays details regarding the selected day (previously selected by the user with the calendar control). The details are:

    • Selected date
    • Weather of the day
    • Sunrise and sunset
    • Temprerature (in Â°F)
    Additionally it contains also a slider that enables the user to select an hour of the day. If the user move the slider automatically the map will display trips that are active at that time and also the percentage of completion for every trip. The trip informations are displayed directly on the map, every bike is a red circle, and its path is marked with a black line. The portion of path already completed is in red. The path is calculated interpolating the initial station and the final station coordinates and estimate the position of the bike at that time. When that user select the hour automatically the weather and the temperature will be adjusted to the nearest measure.

    This controller contains also a button that hides it from the map. When the day controller is hidden also the bikes visualization will be hidden.

    Zoom controller

    This controller enable the user to zoom in and zoom out on the map. In order to provide the best possible user experience there are a minimum and a maximum level of zoom that let the user focus on the Chicago map instead of let him explore the entire world.

    Multi-map button

    This is a button that enable the multimap visualization. During the multimap visualization in the same place will appear a Single-map button that disable multimap visualization.

    Show and hide charts buttons

    This are two buttons that let the user choose which category of graph visualize.

    Filters controlles

    This controller let the user choose how to filters data displayed on the graphs. When the application starts all filters are disabled.

    There are three filters:

    • Gender filer: is composed by two buttons (male/female)
    • Age filer: it is a double slider, one cursor permits to select the minimum age and the other the maximum. This GUI component is specifically designed for this application in order to provide the best user experience and improve usability. The user can move independently the two cursors of the slider, the application will automatically recognize the ages choosen.
    • Filter by user type: there are two type of Dyvvy bikes user: customer: the casual user and the subscriber: the user that buy a subscription.
    Since only subscribers give their personal data the user can't select customer filter and gender filter at the same time (unless he/she accept to see nothing on the graphs).


    Charts

    There are two category of charts:

    • Static charts: displays information about general statistical information about bikes and trips. This informations are not dependent from the date, and for that reason are called static. The user can also apply filters on this graphs using the filters controller.
    • Dynamic charts: displays information about the trips in different formats simultaneusly. This graphs depends on the selected date (if there's one) and on the active filetrs.
