The Data

The main part of a Data Driven Document

Data selection

The aim of this project is displaying on a big screen data that represent Divvy bikes trips taken between July and December 2013. Since there was a huge amount of data I selected the most important. I needed data about all the trips, at least when they started, ended, from where and to where. Sinche the data contains also the demographic data for registered users (without a user identification for provacy) I also extracted and incorporated it in my data representation. In order to represent th estations I needed the information about the location (latitude and longitude coordinates) and about the launch data (that was very useful for calculating the popularity). For representing data I used a MySQL database, and for that reason all data has a formal representation described using SQL language. Since I don't have demographic data for all trips NULL values are allowed.

Data structure

In order to insert the data inside the DB I needed a formal SQL structure. In order to give the maximum performance without sacrificing the data integrity I created three different tables linked with foreign keys: TRIP, STATION and WEATHER. Since the data where already in CSV format I only needed to import them using the import function of Sequel pro and map every field on the correspondig MySQL table field.
